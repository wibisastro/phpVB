# Release Notes — v5.0.0-alpha

**Tanggal:** 28 Februari 2026
**Fase:** 1 dari 7 (Foundation — PHP 8.3 & Tooling)
**Backward Compatible:** Ya — semua fungsionalitas existing tetap berjalan

---

## Ringkasan

Rilis ini merupakan langkah pertama dari seri refactoring besar phpVB/Gov2Framework menuju arsitektur modern. Fokus pada upgrade PHP runtime ke 8.3, modernisasi syntax seluruh codebase, penambahan Enums dan typed Exceptions, serta setup infrastruktur testing dan code quality tools.

Tidak ada perubahan fungsionalitas. Semua behavior aplikasi tetap sama — hanya fondasi kode yang dimodernisasi.

---

## Perubahan

### PHP Runtime & Dependencies

- **PHP platform** dinaikkan dari `8.1` ke `^8.3`
- **guzzlehttp/guzzle** `~6.0` → `^7.8` (major upgrade, Guzzle 6 sudah deprecated)
- **mailgun/mailgun-php** `~1.7.2` → `^4.0`
- **nikic/fast-route** `^1.2` → `^1.3`
- **league/oauth2-client** `^2.6` → `^2.7`
- **Ditambahkan:** `vlucas/phpdotenv ^5.6` — untuk environment variable management via `.env`
- **Dihapus:** `asimlqt/php-google-spreadsheet-client` — deprecated, tidak lagi digunakan

### Dev Dependencies Baru

- `phpunit/phpunit ^11.0` — testing framework
- `phpstan/phpstan ^2.0` — static analysis
- `friendsofphp/php-cs-fixer ^3.0` — code style fixer
- `mockery/mockery ^1.6` — mocking library

### PHP Enums (Baru)

Empat enum baru di `core/lib/Enums/`:

- **UserRole** — `public`, `guest`, `pimpinan`, `member`, `admin`, `webmaster`, `owner` dengan method `level()`, `hasPrivilege()`, `fromLegacy()`, dan `toLegacyArray()` untuk interop dengan kode lama
- **UserStatus** — `active`, `pending`, `suspended`, `inactive` dengan method `canAccess()` dan `blockMessage()`
- **HttpMethod** — `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`
- **NotificationType** — mapping antara Bulma class names (`is-primary`, `is-danger`) dan Bootstrap 5 variants, dengan method `toBootstrap()` dan `fromLegacy()`

### Typed Exceptions (Baru)

Tujuh exception class baru di `core/lib/Exceptions/`:

- **HttpException** (base) — menyimpan status code, memiliki `toLegacyFormat()` untuk backward compat dengan format `"Code:Message"` lama
- **AuthenticationException** — 401, default message "Halaman ini membutuhkan login"
- **AuthorizationException** — 403, default message "Anda tidak memiliki wewenang..."
- **ValidationException** — 422, menyimpan array errors per-field
- **NotFoundException** — 404
- **DatabaseException** — 500
- **ConfigException** — 500

### Modernisasi Syntax (19 files core/lib)

Seluruh file di `core/lib/` telah di-refactor:

- Visibility modifiers (`public`/`private`/`protected`) ditambahkan ke semua method
- Parameter type hints dan return type declarations di semua function
- `match` expression menggantikan `switch` statement di tempat yang sesuai
- `str_contains()`, `str_starts_with()`, `str_ends_with()` menggantikan `substr()`, `strstr()`
- Array syntax `[]` menggantikan `array()`
- Null safe operator `?->` dan null coalescing `??` diterapkan
- PHPDoc blocks lengkap di semua class dan method
- Closing `?>` tags dihapus (PSR-12)
- Dead code dan commented-out code dibersihkan

### Helpers Baru

- `safe_string(mixed $value, string $default)` — safe cast ke string
- `cint()` — ditambahkan `mixed` type hint pada parameter

## Autoload

PSR-4 autoload sudah mencakup Enums dan Exceptions secara otomatis:

```
Gov2lib\               → core/lib/
Gov2lib\Enums\         → core/lib/Enums/
Gov2lib\Exceptions\    → core/lib/Exceptions/
Tests\                 → tests/
```

Helper functions di-load via `files` autoload di composer.json.

## Catatan Upgrade

### Untuk Developer

1. Jalankan `composer update` setelah pull perubahan ini
2. Pastikan PHP runtime >= 8.3 terinstall di local dan server
3. Jika menggunakan Laragon, update PHP ke 8.3 di Settings
4. Jika menggunakan Ubuntu:
   ```bash
   sudo apt install php8.3 php8.3-{common,cli,fpm,mbstring,xml,zip,curl,gd,mysqli}
   ```
5. Verifikasi dengan `composer check`

### Breaking Changes

Tidak ada breaking changes pada level fungsionalitas aplikasi. Namun:

- **Guzzle 7** memiliki beberapa perubahan API dari Guzzle 6 — jika ada kode custom yang menggunakan Guzzle langsung di luar framework, perlu disesuaikan
- **Mailgun SDK v4** memiliki API yang berbeda — jika ada pengiriman email custom di luar framework, perlu dicek

### Yang Tidak Berubah

- Alur routing tetap sama
- Semua XML config tetap dibaca seperti biasa
- Session JWT tetap menggunakan cookie `Gov2Session`
- MeekroDB query patterns (`\DB::query()`, `\DB::insert()`, dll) tetap berfungsi
- Template Twig tetap di-render seperti biasa
- Vue components tetap di-load via httpVueLoader
